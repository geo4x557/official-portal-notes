# official-portal-notes

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文件形式存放，适用于信息展示、参考记录或轻量级门户用途。

## 项目简介

`official-portal-notes` 是一个以 HTML 页面为主要内容的归档仓库。每个页面均为独立文件，不依赖后端服务或数据库，可直接在浏览器中打开查看。仓库结构简单，便于维护和扩展。

## 目录说明

```
official-portal-notes/
├── pages/          # 存放所有独立 HTML 页面
│   ├── example1.html
│   └── example2.html
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── index.html      # 首页或导航页（可选）
└── README.md       # 本文件
```

- `pages/`：核心目录，每个 HTML 文件对应一个独立页面，文件名建议使用有意义的英文或拼音。
- `assets/`：存放页面中引用的样式、脚本或媒体资源，按需组织子目录。
- `index.html`：可作为仓库的入口页面，提供页面列表或导航链接（非必须）。

## 页面归档说明

- 所有页面均为纯前端静态 HTML，支持现代浏览器。
- 页面间相互独立，不共享状态或变量。
- 归档内容不针对任何特定域名或网站，仅作为通用资料保存。
- 页面中不应包含外部依赖（如 CDN 库）或指向未知站点的链接，如有需要请将资源本地化至 `assets/`。

## 维护说明

- 添加新页面：在 `pages/` 目录下创建 `.html` 文件，并确保其内部引用资源路径正确。
- 更新现有页面：直接修改对应文件，注意保持代码整洁和注释清晰。
- 删除页面：移除文件后，如有导航页面请同步更新链接。
- 建议每次提交时附带简短说明，标注新增、修改或删除的内容。
